# Shopify App Store — Screenshot Capture Plan

Shopify allows **6-10 screenshots** on the listing. Order matters — the first 3 are visible without scrolling on most screens, so they need to do the heaviest pitching.

## Format requirements (per Shopify guidelines)

- **Recommended size:** 1600 × 900 px (16:9). Acceptable: 1280 × 800.
- **Format:** PNG or JPG. PNG preferred for crisp UI captures.
- **No mockups required** — actual app screenshots are better. Avoid stock photos.
- **Annotations welcome** — you can label specific UI elements, but keep them subtle.

## Suggested annotation style

- Use **blue arrows + a 1-line caption** for any callout
- Caption color: `#0b1220` on a white pill `#fafaf7` with `#e5e7eb` border
- Arrow color: `#2563eb` (the brand primary)
- Font: Inter, 16-18px bold

You can do all 10 in Figma in ~1 hour. Or use a tool like CleanShot X / ShareX that captures + annotates in one pass.

- Arrow at the left-side block-type icons → "Type-specific blocks: heading, list, table, FAQ, gallery…"

## Production tips

- **Use a real article**, not lorem ipsum. The fragrance-focused content from your existing blog reads as authentic and gives reviewers something specific to look at.
- **Clean up sample data** before capturing — make sure no test articles are titled "asdfasdf" or "TEST".
- **Browser chrome:** capture at a clean width (1440px window is ideal), no extra tabs/extensions visible. Use a Chrome profile dedicated to this.
- **Dark mode:** capture in **light mode** — Shopify's listing UI assumes light-mode app screenshots.
- **Privacy:** before publishing, check that no real customer data, real emails, or PII shows in any frame. The hero alt-text Vision example shouldn't have a recognizable real face.
